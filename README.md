# Лекция «2026 AI» — презентация-сайт

Презентация для студентов-вебинженеров (стек C#/.NET) про то, как сейчас
выглядит разработка с AI, кем нужно становиться и что учить дальше.

## Как запустить

**Самый простой способ** — открыть файл `index.html` двойным кликом
в браузере (Chrome / Edge / Firefox). Никакого сервера и интернета не нужно,
всё работает локально.

Если вдруг слайды не подгрузились (редко, на отдельных настройках браузера),
поднимите простой локальный сервер из этой папки:

```powershell
# вариант с Python
python -m http.server 8080
# затем открыть http://localhost:8080
```

## Управление

| Клавиша | Действие |
|---------|----------|
| `→` / `Пробел` / клик в правую половину | следующий слайд |
| `←` / клик в левую половину | предыдущий слайд |
| `N` | показать/скрыть **заметки спикера** (реплики, что говорить вслух) |
| `O` | карта всех разделов — клик переводит на нужный слайд |
| `F` | полноэкранный режим |
| `Home` / `End` | в начало / в конец |
| `Esc` | закрыть карту разделов |

Текущая позиция сохраняется в адресной строке (`#номер`) — можно обновлять
страницу или кидать ссылку на конкретный слайд.

## Структура

- `index.html` — движок презентации и стили (самодостаточный).
- `slides.js` — **всё содержание** лекции. Тут правится текст слайдов.
- `covers/` — обложки книг для слайда «Что почитать» (скачаны локально,
  работают офлайн).
- `lecture_content.md` — исходные материалы (методический каркас + черновики),
  если захочется что-то расширить.

## Разделы

1. Где сейчас AI · 2. Новая роль инженера · 3. Последствия (overqualified) ·
4. Где и что учить · 5. Как мы работаем в компании · 6. Дорожные карты (C# + AI) ·
7. Сквозная задача (API + PostgreSQL) · 8. Практика «собери свой X» + что почитать ·
Финал.

Раздел 8 построен по принципу **build your own X**
([github.com/codecrafters-io/build-your-own-x](https://github.com/codecrafters-io/build-your-own-x)):
студент сначала решает задачу сам до уровня, который ему нравится, потом даёт
ту же задачу AI-агенту и сравнивает — так вырабатывается вкус к хорошему коду
и навык ревью.

## Что нужно дописать перед показом

По тексту встречаются жёлтые плашки **«⚠ Уточнить у преподавателя»** — это
места, где содержание зависит от того, что принято именно в вашей компании/курсе.
Их стоит заполнить своими данными:

1. **PAF framework** (раздел 6, AI-roadmap) — аббревиатура в индустрии
   неоднозначна, поэтому расшифровка намеренно не выдумана. Впишите точное
   определение, которое используете вы.
2. **Hives / «рой» агентов** — устоявшийся термин или внутреннее название?
   Дайте студентам точное определение и, если есть, конкретный инструмент.
3. **Локальные модели и раннеры** — какие именно используете (названия).
4. **Skills** — если есть свой набор скиллов, показать как пример.
5. Мелочи: срок жизни feature flags, порог покрытия тестами, нужность
   PgBouncer, источник пользователей (свой / SSO), конкретный CI-инструмент.

Чтобы поправить любой слайд — откройте `slides.js`, найдите нужный объект
по заголовку (`title`) и отредактируйте `bullets` / `note` / `callout`.

## Тайминг

Ориентир: ~90–110 минут, ~87 слайдов. Разделы 5, 6 и 8 самые плотные — при
нехватке времени их можно проходить тезисно (буллеты — на экран, детали — в
заметках спикера), а основной фокус держать на разделах 2–3 и сквозной
задаче (раздел 7).
